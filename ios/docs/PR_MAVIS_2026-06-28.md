# fix(ios): critical correctness, accessibility baseline, UX polish (big bang)

Addresses the Critical + UX + accessibility items from the deep iOS review at `ios/docs/REVIEW_MAVIS_2026-06-28.md`. 7 commits, +414/-287 lines, 11 files.

## What this fixes

### Critical (correctness / App Store blockers)

| ID | Fix |
|----|-----|
| **C1** | Removed `.id(appearanceMode)` from `ContentView` — was wiping the entire view tree (nav stacks, sheet state) on every theme toggle. Just `.preferredColorScheme()` is enough. |
| **C2** | Moved URLCache bump out of `MBGCApp.init()` into a dedicated `ImageCache` URLSession. No more mutating the global `URLCache.shared` — leaves BGG XML, future telemetry, and other sessions alone. |
| **C3** | `BGGXMLParser` now strips HTML tags (`<br/>`, `<i>`, `<b>`, `<a>`, etc.) from game descriptions in addition to entity unescaping. Collapses whitespace. No more raw HTML rendered in `GameDetailView`. |
| **C4** | BGG XML `<poll>` state leak fixed. Unknown/missing poll name now resets the parser's `currentPollName` so a poll without a `name` attribute doesn't inherit a previous poll's identity. |
| **C5** | Skip BGG `<item>` entries with `id <= 0`. Such rows used to push a `BGGGame` with `bggId=0` which collided with `Game`'s `@Attribute(.unique)` constraint and broke SwiftData inserts. |
| **C8** | `GameDetailView` shows `—` instead of `0 minutes` when `playtime` is nil. The old rendering looked like a data error. |
| **C9** | `HomePillView` active segment uses `Color.accentColor` (tinted) instead of `Color(.label)` (pure black/white). Matches iOS segmented-control convention; no longer looks unfinished. |

| **U8** | "More..." → "Show more" (no ellipsis for inline expand; Apple HIG reserves ellipsis for actions that open another screen). |
| **U9** | Bottom CTA: "Add to... N" → "Add to collection" / "In N collection(s)" using `Label` + system icon (`folder.fill` / `plus`). |

### Accessibility (S1)

Added `accessibilityLabel` to every icon-only button:

- ContentView: Settings, Search, New Collection, HomePill segments
- VibesView: Select games, Filters, Sort by, Collection actions
- FinderView: Back chevron
- ImportView: New collection picker

`FilterView` rows were already `HStack(Image + Text)` — VoiceOver combines them automatically, so no changes there.

## Architecture impact

- **New file** `ios/MBGC/Views/CachedAsyncImage.swift` (91 lines) — drop-in `AsyncImage` replacement backed by `ImageCache.shared`. All 7 `AsyncImage` call sites updated.
- **New file** `ios/MBGC/Util/CollectionName.swift` (24 lines) — name sanitization helpers, single source of truth.
- **Deleted** `ios/MBGC/Views/LibraryView.swift` (15 lines) — dead code.

XcodeGen uses `path: MBGC` (recursive), so the two new files will be auto-picked up on the next `xcodegen generate`.

## Verification

- `swiftc -parse -target x86_64-apple-ios17.0-simulator` passes on all 14 changed/new Swift files (sandbox has no iOS SDK so a full `typecheck` couldn't run).
- All three test suites (`BGGXMLParserTests`, `GameFiltersTests`, `SmartRuleTests`) parse cleanly. The XML parser changes are defensive — they strip HTML/handle state better but preserve the existing valid-input path that the tests cover.

**Before merging**, please run on a Mac with Xcode 16+:

```sh
cd ios && xcodegen generate
xcodebuild test -scheme MBGC \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro'
```
